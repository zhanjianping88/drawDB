# drawDB Editor Integration

The live editor under `/app` is powered by a built static build of the upstream `drawdb-io/drawdb` application.

## Current integration shape

- Marketing site and SEO pages stay in Next.js.
- The real drawDB editor is served as static assets from `public/drawdb-editor`.
- The `/app/[[...slug]]` route embeds that editor in an iframe and forwards path/query state through the hash route.
- Template detail pages open `/app/editor/templates/blank?templateSlug=<slug>`.
- Tool pages open `/app/editor/templates/blank?mode=blank&database=<db>`.
- Template JSON is generated by `src/app/api/editor-template/[slug]/route.ts`.

## Important files

- `src/app/app/[[...slug]]/page.tsx`
- `src/app/api/editor-template/[slug]/route.ts`
- `src/lib/editor/drawdb-template-presets.ts`
- `src/lib/editor/links.ts`
- `public/drawdb-editor/*`

## Rebuild the embedded editor

1. Clone `https://github.com/drawdb-io/drawdb`.
2. Apply the local integration changes:
   - use `HashRouter` in `src/App.jsx`
   - support `templateSlug` and `mode=blank` loading in `src/components/Workspace.jsx`
   - build with `base: "./"` in `vite.config.js`
3. Run:

```bash
npm install
npm run build
```

4. Copy the build output into:

```bash
public/drawdb-editor
```

## Notes

- This keeps the editor isolated from the Next.js React tree, which avoids dependency and routing conflicts.
- The tradeoff is that upstream upgrades require rebuilding the static editor bundle.
